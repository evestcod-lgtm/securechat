# SecureChat — сборка APK + сервер в Termux

## Что тут лежит
- `server.js`, `package.json`, `.env.example`, `start-termux.sh` — сервер, который крутится в Termux на твоём телефоне.
- `www/` — то же самое приложение (html/css/js), упакованное Capacitor'ом в APK.
- `capacitor.config.json` — настройка Capacitor-обёртки.
- `patches/` — нативные Android-файлы (разрешения, фоновый сервис), которые CI сам вставляет в проект при сборке.
- `.github/workflows/build-apk.yml` — сборка APK на GitHub Actions.

Ничего из твоих файлов не удалено и не урезано — только добавлены куски.

---

## 1. GitHub Secrets — куда вставлять ключи Xirsys и адрес сервера

Заходишь в свой репозиторий → **Settings → Secrets and variables → Actions → New repository secret**.
Создаёшь секреты (названия подогнаны точно под поля в кабинете Xirsys):

| Имя секрета                  | Что туда вставить                                              |
|-------------------------------|------------------------------------------------------------------|
| `XIRSYS_API_KEY`             | поле **API Key** в кабинете Xirsys                                |
| `XIRSYS_CHANNEL_CREDENTIALS` | поле **Channel Credentials** в кабинете Xirsys                    |
| `XIRSYS_CHANNEL`             | имя твоего **Channel** (канала), которое ты сам задал при создании|
| `SERVER_URL`                 | адрес твоего сервера, например `http://100.x.x.x:3000` (Tailscale) или `https://xxxx.lhr.life` |

Всё это ты найдёшь в кабинете Xirsys → твой аккаунт → твой канал.

CI сам сходит в Xirsys API при каждой сборке, получит актуальные ICE-сервера (TURN/STUN) и подставит их вместе с адресом сервера в `www/config.js` — в репозитории эти ключи никогда не лежат в открытом виде.

**Если задашь `SERVER_URL`** — APK при первом запуске сразу подключится к нужному серверу, ничего вводить не придётся.
**Если `SERVER_URL` не задать** — приложение спросит адрес один раз при первом запуске (окно с полем ввода) и запомнит его.

Если Xirsys-секреты не заданы — приложение всё равно соберётся и будет звонить через бесплатный Open Relay TURN (запасной вариант, уже встроен в код).

### Подпись APK (чтобы APK не был debug-версией) — тоже секреты, по желанию
| Имя секрета         | Что туда вставить |
|---------------------|--------------------|
| `KEYSTORE_BASE64`   | твой keystore-файл, закодированный в base64 (команда ниже) |
| `KEYSTORE_PASSWORD` | пароль хранилища |
| `KEY_ALIAS`         | алиас ключа |
| `KEY_PASSWORD`      | пароль ключа |

Как сделать keystore и закодировать (один раз, на своём компе с Java):
```bash
keytool -genkey -v -keystore release.keystore -alias securechat -keyalg RSA -keysize 2048 -validity 10000
base64 -w0 release.keystore > release.keystore.b64   # содержимое этого файла — в KEYSTORE_BASE64
```
Без этих секретов CI просто соберёт debug-APK — он тоже нормально ставится и работает на телефоне, просто не годится для публикации в Play Store (тебе это и не нужно, ты собираешь для сайдлоада).

## 2. Как собрать APK

1. Залей всё содержимое этого архива в свой GitHub-репозиторий (поверх текущих файлов).
2. Добавь секреты Xirsys (см. выше). Подпись — по желанию.
3. Зайди во вкладку **Actions** репозитория → workflow **"Build SecureChat APK"** → **Run workflow** (или просто запушь в `main` — соберётся само).
4. Через несколько минут в конце сборки появится артефакт **SecureChat-APK** — скачай его, внутри `.apk` файл.
5. На телефоне: разреши установку из неизвестных источников (Настройки → Приложения → Особый доступ → Установка неизвестных приложений → выбери свой файловый менеджер/браузер → разреши), открой скачанный APK → установи.

## 3. Сервер в Termux — запуск на телефоне

```bash
pkg update && pkg upgrade -y
pkg install nodejs git -y
termux-setup-storage

# зайди в папку с проектом (например, после git clone своего репозитория)
cd securechat
npm install

cp .env.example .env
nano .env     # впиши свой ADMIN_PASSWORD и XKEY (любая длинная случайная строка)

chmod +x start-termux.sh
./start-termux.sh
```

Чтобы Termux не убивал процесс, когда гаснет экран:
- выполни `termux-wake-lock` (уже встроено в `start-termux.sh`)
- в настройках Android → Приложения → Termux → Батарея → **без ограничений** (отключи оптимизацию батареи)
- поставь из F-Droid **Termux:Boot**, если хочешь, чтобы сервер поднимался сам после перезагрузки телефона

## 4. Чтобы работало не только на одном Wi-Fi (звонки и сообщения из любой сети)

Тут две разные вещи, обе нужны:

**А) Чтобы APK вообще "видел" твой сервер из любой сети** (не только домашний Wi-Fi):
Локальный IP телефона (`192.168.x.x`) снаружи домашней сети недоступен. Проще и надёжнее всего поставить **Tailscale** — бесплатный VPN, даёт телефону-серверу постоянный приватный IP (`100.x.x.x`), который виден с телефона-клиента из любой сети (мобильный интернет, другой Wi-Fi и т.д.), без пробрасывания портов на роутере.

1. Установи Tailscale на телефон-сервер (в Termux: `pkg install tailscale`, либо приложение Tailscale из Play Store) и авторизуйся.
2. Установи Tailscale (приложение из Play Store) на телефон(ы)-клиенты, авторизуйся тем же аккаунтом.
3. Узнай Tailscale-IP сервера: `tailscale ip -4`.
4. Впиши этот адрес (в формате `http://100.x.x.x:3000`) в секрет **`SERVER_URL`** в GitHub (см. пункт 1) и пересобери APK — тогда приложение само подключится, ничего вводить не надо. Либо, если не хочешь пересобирать APK каждый раз при смене адреса, оставь `SERVER_URL` пустым — приложение спросит адрес один раз при первом запуске и запомнит его.

**Б) Чтобы аудио/видео звонок реально соединялся, когда оба телефона в разных сетях (Wi-Fi ↔ 4G):**
Это уже решает Xirsys TURN (пункт 1) — как только ICE-сервера подставлены из Xirsys, WebRTC сам пробьёт NAT между разными сетями. Без TURN P2P-звонок между разными сетями часто не устанавливается — поэтому Xirsys (или встроенный бесплатный Open Relay как запасной) обязателен.

## 5. Разрешения — что и как приложение спросит

В `AndroidManifest.xml` (патчится автоматически при сборке) объявлены:
- `CAMERA`, `RECORD_AUDIO` — для видео/аудио звонков
- `POST_NOTIFICATIONS` — для уведомлений (Android 13+)
- `VIBRATE` — вибрация на входящий звонок/сообщение
- `WAKE_LOCK`, `FOREGROUND_SERVICE`, `FOREGROUND_SERVICE_DATA_SYNC` — чтобы приложение не засыпало и ловило события в фоне

При первом запуске приложение (нативный код `MainActivity.java`) сразу запрашивает системные диалоги на камеру/микрофон/уведомления. Плюс, когда включаешь звонок, `WebView` автоматически отдаёт странице доступ к камере/микрофону (без второго дублирующего системного окна).

## 6. Уведомления и сохранённые звуки

- Рингтон и звук уведомлений, которые ты уже сохранял (через "загрузить свой звук" в настройках) — хранятся в `localStorage` телефона, это никуда не делось и продолжает работать так же и в APK, и на сервере в браузере.
- Добавлены **локальные уведомления** (`@capacitor/local-notifications`): когда приложение свёрнуто, на новое сообщение или входящий звонок вылезет системное уведомление Android с вибрацией — не только звук внутри открытого приложения.
- Фоновый сервис (`KeepAliveService`) держит процесс приложения живым, чтобы сокет не отваливался, пока экран выключен — уведомления приходят даже когда ты не смотришь в телефон. Это не полноценный push через Firebase (для этого нужен отдельный аккаунт Google/Firebase) — если Android всё же выгрузит приложение из памяти при агрессивной экономии батареи, добавь SecureChat в исключения автозапуска/батареи в настройках прошивки телефона (у MIUI/ColorOS/OneUI это отдельный пункт «Автозапуск» помимо стандартной оптимизации батареи).

## 7. Проверка перед использованием

Обязательно после первой сборки:
1. Установи APK, при первом запуске введи Tailscale/локальный адрес сервера.
2. Разреши все системные запросы (камера/микрофон/уведомления) — если случайно нажал "Запретить", зайди в Настройки Android → Приложения → SecureChat → Разрешения и включи вручную.
3. Залогинься тем же аккаунтом на сервере (Termux) и в APK — проверь чат и звонок между двумя устройствами, одно из которых на мобильном интернете (не на домашнем Wi-Fi), чтобы убедиться, что TURN и Tailscale реально работают "не в одной сети".
4. Проверь, что после выключения экрана на 2-3 минуты уведомление о новом сообщении всё ещё приходит.
