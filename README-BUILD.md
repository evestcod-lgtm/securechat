# SecureChat — сборка APK + сервер в Termux (с автообновлением адреса)

## Что тут лежит
- `server.js`, `package.json`, `.env.example`, `start-server.sh`, `start-tunnel.sh` — сервер и Cloudflare-туннель, которые крутятся в Termux на твоём телефоне.
- `www/` — то же приложение (html/css/js), упакованное Capacitor'ом в APK. Внутри `boot.js` — загрузчик, который перед стартом узнаёт актуальный адрес сервера.
- `capacitor.config.json` — настройка Capacitor-обёртки.
- `patches/` — нативные Android-файлы (разрешения, фоновый сервис) + скрипты автозапуска и автообновления адреса сервера.
- `.github/workflows/build-apk.yml` — сборка APK на GitHub Actions.

Ничего из твоих исходных файлов не удалено — только добавлено.

---

## Как это устроено (коротко)

Ссылка на сервер (Cloudflare Quick Tunnel) меняется при каждом перезапуске туннеля. Поэтому адрес **не зашивается в APK** и не хранится в GitHub Secret — вместо этого:

1. Termux при каждом запуске туннеля сам ловит свежую ссылку и записывает её в открытый файл `patches/current-server-url.txt` в твоём же GitHub-репозитории (через GitHub API) — полностью автоматически, руками ничего вставлять не нужно.
2. Приложение (APK) при каждом запуске сначала скачивает этот файл с GitHub (доля секунды) и уже потом подключается по адресу, который там написан.

Тебе никогда не нужно вручную вводить адрес или пересобирать APK при смене ссылки — только держать `start-tunnel.sh` запущенным (или включить автозапуск, см. пункт 3).

## 1. Секреты и токены — что и куда

### GitHub Secrets (Settings → Secrets and variables → Actions → New repository secret)

Для сборки APK (Xirsys — звонки между разными сетями):

| Имя секрета                  | Что туда вставить                                |
|-------------------------------|----------------------------------------------------|
| `XIRSYS_API_KEY`             | поле **API Key** в кабинете Xirsys                  |
| `XIRSYS_CHANNEL_CREDENTIALS` | поле **Channel Credentials** в кабинете Xirsys      |
| `XIRSYS_CHANNEL`             | имя твоего **Channel** в Xirsys                     |

`SERVER_URL` как секрет **больше не нужен** — адрес теперь берётся автоматически из файла в репозитории.

Если Xirsys не заполнен — приложение соберётся и будет работать через бесплатный запасной TURN (Open Relay), просто послабее.

### GitHub Personal Access Token (нужен для автообновления адреса — хранится только на телефоне, не в GitHub Secrets)

1. github.com → твоя фотка (справа сверху) → **Settings**
2. Внизу слева **Developer settings**
3. **Personal access tokens → Tokens (classic) → Generate new token (classic)**
4. Имя — любое, например `securechat-url-updater`
5. Срок действия — **No expiration**
6. Отметь галочку **repo**
7. **Generate token**, скопируй значение (начинается с `ghp_`)

Этот токен вписывается в `.env` **на телефоне** (см. ниже), в GitHub Secrets он не нужен.

## 2. Сервер в Termux — установка и первый запуск

```bash
pkg update && pkg upgrade -y
pkg install nodejs git cloudflared -y
```

Скачай проект (или перекинь файлы вручную):
```bash
git clone https://github.com/evestcod-lgtm/securechat.git
cd securechat
npm install
```

Настрой `.env`:
```bash
cp .env.example .env
nano .env
```
Впиши:
- `ADMIN_PASSWORD` — свой пароль администратора
- `XKEY` — любая длинная случайная строка
- `GITHUB_TOKEN` — токен из шага выше
- `GITHUB_REPO` — уже стоит `evestcod-lgtm/securechat`, поменяй только если репозиторий другой

Сохрани: `Ctrl+O` → `Enter` → `Ctrl+X`.

Запусти сервер и туннель:

```bash
chmod +x start-server.sh start-tunnel.sh patches/*.sh
./start-server.sh
./start-tunnel.sh
```

`start-tunnel.sh` сам поймает свежую ссылку Cloudflare и отправит её на GitHub, ты увидишь:
```
✅ Твой адрес сервера: https://случайные-слова.trycloudflare.com
```

После этого можно закрывать сессию Termux — туннель и сервер продолжат работать в фоне (запущены через `nohup`).

## 3. Автозапуск при включении телефона (чтобы не запускать руками каждый раз)

1. Установи **Termux:Boot** — из F-Droid (не Play Store), приложение называется именно так. Если нет F-Droid — сначала поставь f-droid.org, потом через него Termux:Boot.
2. Открой Termux:Boot один раз (просто открыть-закрыть, чтобы Android его зарегистрировал).
3. В Termux:
```bash
mkdir -p ~/.termux/boot
cp ~/securechat/patches/termux-boot-start-securechat.sh ~/.termux/boot/start-securechat.sh
chmod +x ~/.termux/boot/start-securechat.sh
```

Теперь при каждом включении/перезагрузке телефона сервер и туннель поднимутся сами, поймают новую ссылку и обновят её на GitHub — без твоего участия.

**Обязательно отключи оптимизацию батареи для Termux:**
Настройки Android → Приложения → Termux → Батарея → без ограничений.
Иначе Android может убить процесс, и туннель оборвётся раньше времени.

## 4. Сборка APK

1. Залей всё содержимое этого архива в свой GitHub-репозиторий (если ещё не сделал).
2. Добавь секреты Xirsys (см. пункт 1). Подпись APK — по желанию, см. ниже.
3. Actions → "Build SecureChat APK" → **Run workflow** → жди 5–10 минут.
4. Скачай артефакт **SecureChat-APK** внизу завершённого запуска.
5. На телефоне: разреши установку из неизвестных источников, открой скачанный `.apk`, установи.

APK нужно собрать **только один раз** — дальше он сам всегда находит актуальный сервер через `boot.js`, пересобирать при смене ссылки не нужно.

### Подпись APK (по желанию)
| Имя секрета         | Что туда вставить |
|---------------------|--------------------|
| `KEYSTORE_BASE64`   | твой keystore-файл, закодированный в base64 |
| `KEYSTORE_PASSWORD` | пароль хранилища |
| `KEY_ALIAS`         | алиас ключа |
| `KEY_PASSWORD`      | пароль ключа |

```bash
keytool -genkey -v -keystore release.keystore -alias securechat -keyalg RSA -keysize 2048 -validity 10000
base64 -w0 release.keystore > release.keystore.b64
```
Без этого CI соберёт debug-APK — он тоже нормально ставится и работает.

## 5. Разрешения приложения

Патчатся автоматически при сборке (`patches/manifest-permissions.xml` + `patches/MainActivity.java`):
- `CAMERA`, `RECORD_AUDIO` — видео/аудио звонки
- `POST_NOTIFICATIONS` — уведомления (Android 13+)
- `VIBRATE` — вибрация на звонок/сообщение
- `WAKE_LOCK`, `FOREGROUND_SERVICE`, `FOREGROUND_SERVICE_DATA_SYNC` — чтобы приложение не засыпало и ловило события в фоне

При первом запуске APK сразу спросит камеру/микрофон/уведомления системными диалогами.

---

## 6. Уведомления и сохранённые звуки

- Рингтон и звук уведомлений, которые ты сохранял через "загрузить свой звук" — в `localStorage`, продолжают работать и в APK.
- Локальные уведомления (`@capacitor/local-notifications`) — на новое сообщение/звонок вылезет системное уведомление Android с вибрацией, даже если приложение свёрнуто.
- `KeepAliveService` держит процесс живым в фоне, чтобы сокет не отваливался при выключенном экране. Если прошивка телефона агрессивно "чистит" память (MIUI/ColorOS/OneUI) — добавь SecureChat в разрешённые автозапуски отдельно от обычной оптимизации батареи.

## 7. Проверка

1. Сервер запущен в Termux (`./start-server.sh` + `./start-tunnel.sh` в двух сессиях, или автозапуск после перезагрузки).
2. Установи и открой APK — должен подключиться сам, без запроса адреса.
3. Разреши системные запросы (камера/микрофон/уведомления).
4. Проверь чат и звонок с телефона на мобильном интернете (не домашнем Wi-Fi), чтобы убедиться что TURN (Xirsys/Open Relay) реально пробивает разные сети.
5. Перезагрузи телефон, подожди минуту — проверь, что сервер и туннель поднялись сами (`cat ~/securechat/autostart.log`), а APK при следующем открытии всё ещё подключается (даже если ссылка сменилась).
